# Beauty Hub Visual Lock

## Primary approved direction

**Dark Polished Beauty Editorial** is the main and final visual direction for Beauty Hub.

Beauty Hub must feel like a strict dark editorial archive / private beauty club for beauty masters, not like a technological dashboard, terminal, SaaS product, crypto interface, HUD, cyber system, or generic beauty landing.

The site is a route system, but its visual language must not look technological. The system must be expressed through structure, navigation, rooms, cards, editorial hierarchy, art direction and conversion paths.

## Core feeling

The approved feeling:

- dark polished editorial archive;
- private beauty club;
- matte graphite;
- polished grey luxury;
- dusty white / ivory surfaces;
- aged copper details;
- rare bitter orange as light, not as cheap button color;
- monumental typography;
- quiet spacing;
- sculptural cards;
- rooms as separate editorial halls;
- Telegram as closed access into the living project.

The direction can be Rick Owens-adjacent in mood: dark asceticism, monolithic forms, raw luxury, anti-gloss, sculptural restraint. It must not copy fashion branding or become gothic for the sake of style.

## Absolute visual bans

Do not reintroduce:

- technological dashboard aesthetics;
- SaaS UI;
- crypto / fintech terminal aesthetics;
- HUD / cyber interface;
- computer database UI;
- terminal UI as the main look;
- neon;
- cyberpunk;
- scanlines, glitch, hacking/loading UI;
- concrete buildings;
- grey concrete wall backgrounds;
- brutalist building placeholders;
- architecture photo backgrounds;
- graffiti / street art;
- pink salon beauty templates;
- glossy infobusiness luxury;
- fake metrics;
- fake testimonials;
- random 3D;
- glow/blob gradients as decorative filler.

Important: do not use concrete as the main visual idea. If texture is needed, use matte graphite, dark mineral surface, polished dark stone, dusty editorial grain, ivory paper or marble-like warmth. No grey concrete wall, building, facade, tunnel or architecture placeholder.

## Concept art layer

The site must not be only dark layout plus text and cards. It needs a coherent art layer that connects Hero, route cards, rooms, Telegram and final CTA.

The art layer principle:

**Light works as navigation, not decoration.**

Use warm controlled orange light as:

- rim light on a dark object;
- reflection on matte graphite / metal / ivory;
- a thin route line;
- a soft glow from depth;
- a directional studio light edge;
- a subtle hover light on a card.

Do not use orange as large flat UI fill everywhere. Bright orange buttons alone look cheap. Orange should guide attention through light, reflection and edge accents. Bitter orange remains reserved for important CTA moments.

## Hero direction

Hero must be the strongest visual scene.

It should feel like a dark fashion-editorial cover or entrance into a private club.

Hero visual:

- dark graphite space;
- one large abstract sculptural form, not a building;
- matte black stone / graphite slab / dense dark fabric / route object / archive plate feeling;
- orange rim light along one edge;
- very subtle mineral grain;
- large typography with air;
- no tech grid if it makes the scene technological.

Hero must not use:

- building photo;
- concrete facade;
- urban architecture;
- dashboard panel;
- terminal screen;
- HUD;
- neon;
- orange glow blob;
- salon photo;
- random manicure photo as the main background.

Hero gives emotional entry. The next sections must become calmer, cleaner and more functional.

## Cards direction

Route cards are physical-feeling editorial passes, not information boxes.

They must feel like:

- private access passes;
- fashion archive covers;
- route tags;
- matte club cards;
- object cards on graphite or ivory surfaces.

Homepage card should stay short:

```text
01
TECHNIQUE

Техника / материалы / скорость / чистота

ОТКРЫТЬ МАРШРУТ →
```

No long signal / first step / breakdown on homepage cards. Put details in Route Hall or route rooms.

Card visual rules:

- large room number;
- short route title;
- one short descriptor line;
- a lot of air;
- thin aged copper line;
- subtle orange rim light or inner edge light;
- CTA as access action, not loud sales button;
- almost no shadow;
- no heavy glow;
- no technical frame clutter.

## Rooms direction

Each route room is a separate editorial hall inside one private archive.

### Technique

Surgical beauty precision. Clean, cold, almost medical, but not clinical. Dark graphite, ivory fragments, clean surface, instrument / steel / material reflections, exact lines, orange edge light.

### Client Flow

Movement of a client path. Not dashboard, not metrics. Editorial route map, thin line, touchpoints, quiet markers, horizontal movement, occasional ivory notes.

### Independence

Transition from dense dark zone into more open space. Not motivational exit-to-light cliche. A controlled passage, dark polished plane, narrow orange slit of light, more air on the destination side.

### Mentorship

Private analysis room. Quiet, editorial, expensive, human. Dark graphite, ivory notes, archive sheets, matte glass feeling, large fields, private mark, warm orange reflection in the corner.

## Julia block

Julia must be the human trust center of the strict system.

Portrait = status.
Hands / tools / material = practice.
14 years = authority.
Routes = structure.
Warm light = life.
Ivory = softness.
Graphite = status.

Do not make Julia look like a distant fashion model only. The block must show quiet authority and practice, not arrogance. Use portrait plus working details where possible.

## Ivory / white layer

White is not ordinary white background. It is material:

- dusty white;
- warm ivory;
- matte ivory surface;
- expensive editorial paper;
- subtle marble-like warmth without obvious cheap marble veins;
- calm surface inside dark space.

Use ivory rarely and intentionally: Product System, selected card layers, CTA zones, Mentorship notes, editorial inserts.

Graphite + Ivory + Orange Light is the main premium contrast.

## Navigation

Navigation must feel club-like and clear, not technical.

Every route page must make clear:

- where the user is;
- how to return to Route Hall;
- previous room;
- next room;
- Telegram access;
- product access if relevant.

Mobile should use a bottom dock: Hall / Rooms / Telegram. It must not look like a tech dashboard.

## Motion

Motion must be expensive, calm and non-technological.

Allowed:

- calm card opening;
- smooth room entry;
- thin line passing over a card;
- soft text reveal;
- slight dimming of inactive cards;
- slow button reaction;
- very subtle Telegram access pulse.

Forbidden:

- glitch;
- scanline;
- hacking/loading UI;
- neon pulse;
- cyber transition;
- aggressive parallax;
- scrolljacking;
- bounce;
- infinite rotations.

## Conversion logic

Main shared CTA:

**ВОЙТИ В TELEGRAM**

Product-specific CTA:

- Technique: **ОТКРЫТЬ КУРС ПО ТЕХНИКЕ** + secondary Telegram.
- Mentorship: **ЗАПРОСИТЬ ЛИЧНЫЙ РАЗБОР** + secondary Telegram.
- Client Flow: **РАЗОБРАТЬ КЛИЕНТСКИЙ ПОТОК В TELEGRAM**.
- Independence: **СОБРАТЬ ПЛАН ВЫХОДА / ЗАПРОСИТЬ РАЗБОР**.

Do not turn the site into hard-sell infobusiness. The site should feel like access to a system, while conversion remains clear.

## Final formula

Do not build a dark technology base.
Build a dark editorial route archive.

Do not build a terminal.
Build a closed access point.

Do not build dashboard cards.
Build private passes.

Do not build text pages.
Build separate editorial halls.

Do not build effects.
Build calm expensive interface reactions.

**Dark Polished Beauty Editorial is now the main concept and must override older Brutal Concrete / terminal / tech interpretations.**
